The same goes with Radial Gradianets

Default

Just type "radial-gradient(color1, color2, color3);"

You can change the space of the colors.

Just type "radial-gradient(color1 5%, color2 15%, color3 60%);"

Change the shape.

The default is an oval but you can make it a circle type "linear-gradient(circle, color1, color2, color 3)"

Woah! The gradient seems so big now! Don’t worry, we will learn how to change its size.

The size keywords are

    closest-side
    farthest-side
    closest-corner
    farthest-corner
All of these refer to the center of the gradient and the specified side or corner of the image window. So for example Closest-Side will extend the gradient to the side closet to it but Farthest-Corner will push the gradient all the way to the farthest corner, even if the fest of the gradient overflows from the image window.

Some examples.

This gradient is the default one

This gradient has Closest-Side added to it, so the whole thing does not cross the nearest side, which happens to be the top and bottom. “linear-gradient(closest-side, rgb(0, 255, 225), rgb(17, 17, 21));”

However this one has Farthest-Side added to it, and now it goes all the way to the left and right sides but does not pass them? “linear-gradient(farthest-side, rgb(0, 255, 225), rgb(17, 17, 21));”

It can be a little more obviously if we move the grandaunt away from the middle of the viewing window, like this “linear-gradient(farthest-side at 90% 55%, rgb(0, 255, 225), rgb(17, 17, 21));”

Now you can see the center of the gradient is almost all the way to the right, but the left side of it still stretches all the way to the left side because it was told to expend to the Farthest-Side.

Let’s do the same thing but make sure the gradient it bound to the Closest-Side. Tada! Look how small it is! That's because it went to the closest side and then stopped.
